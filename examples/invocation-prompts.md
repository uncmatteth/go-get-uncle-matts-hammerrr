# Invocation Prompts

## Positive Triggers

- "Go get Uncle Matt's HAMMERRR and audit this repo."
- "Run a project truth audit before we claim this is ready."
- "Check whether the README, install path, tests, and release package match reality."
- "Find stale claims, ignored flags, false-success paths, and proof gaps."

## Near-Miss Prompts

These should usually route somewhere else:

- "Review this PR for bugs."
- "Fix this failing test."
- "Refactor this component."
- "Do a quick style pass on this README."

## Ambiguous Prompts

Ask for scope or default to a smaller audit:

- "Is this project okay?"
- "What is wrong here?"
- "Can we ship this?"
